Core concepts behind the Turing Machine model

📼

The Tape

An infinite sequence of cells, each holding one symbol from the tape alphabet Γ. Initially populated with the input; remaining cells hold blank (B).

🔵

States (Q)

A finite set of internal configurations. Always includes a start state q₀, accept state q_accept, and reject state q_reject.

δ

Transition Function

δ: Q × Γ → Q × Γ × {L, R}. Given the current state and tape symbol, it returns a new state, writes a symbol, and moves the head Left or Right.

✓

Acceptance

A string is accepted if the machine enters q_accept. It is rejected if it enters q_reject or if no transition is defined (implicit reject).

⚙️

Configuration

A snapshot: (state, tape contents, head position). The simulation history is a sequence of configurations, one per step.

∞

Turing-Completeness

A system is Turing-complete if it can simulate any Turing Machine. Modern CPUs and languages are all Turing-complete.

Formal Definition: M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

  • Q — finite set of states
  • Σ — input alphabet (no blank)
  • Γ — tape alphabet (Σ ⊂ Γ, B ∈ Γ)
  • δ — transition function: Q × Γ → Q × Γ × {L,R}
  • q₀ — start state
  • q_accept, q_reject — halting states
